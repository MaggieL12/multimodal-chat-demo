# 技术说明文档 - Talk 2 Mag 多模态聊天系统

## 1. 项目概述

Talk 2 Mag 是一个基于阿里云 Qwen-VL 模型的多模态聊天系统，支持文字和图片输入，具备多轮对话记忆能力。系统采用 Gradio 构建 Web 界面，使用阿里云 OSS 存储用户上传的图片。

**核心功能**：
- ✅ 纯文本对话（多轮记忆）
- ✅ 图片理解（上传并提问）
- ✅ 参数调节（控制回复长度）
- ✅ 公网访问（Gradio 自动生成链接）

## 3. 部署说明

### 3.1 环境要求
- Python 3.9+
- 阿里云账号（开通百炼服务 + OSS服务）
- 网络连接（需访问阿里云 API）

### 3.2 安装步骤
```bash
# 克隆项目
git clone https://github.com/MaggieL12/multimodal-chat-demo.git
cd multimodal-chat-demo

# 创建虚拟环境
python -m venv venv
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt
```

### 3.3 配置密钥
在 `app_new.py` 中替换以下配置：
```python
# 阿里云百炼 API Key
api_key="你的百炼API Key"

# OSS 配置
OSS_ACCESS_KEY_ID = "你的AccessKey ID"
OSS_ACCESS_KEY_SECRET = "你的AccessKey Secret"
OSS_BUCKET_NAME = "talk2mag-media"
```

### 3.4 启动
```bash
python app_new.py
```

启动后访问：

本地地址：http://127.0.0.1:7860

公网地址：系统支持 Gradio 自动生成公网分享链接，如需生成 Gradio 公网链接，需确保网络能访问 huggingface.co 以下载 frpc 文件，或手动安装。

6. 失败案例分析
案例1：视频理解功能探索
尝试过程：

初期使用智谱 AI API，图片功能顺利

为满足“多类型输入”加分项，尝试集成视频理解

切换到阿里云 Qwen-VL 系列模型

配置 OSS 存储，确保视频文件可公网访问

遇到的问题：

| 问题 | 现象 | 原因分析 |
|:---|:---|:---|
| 视频无法理解 | 模型回复“没有提供具体内容” | 视频文件虽可下载，但模型下载失败 |
| 权限问题 | 返回“Failed to download” | OSS 文件未设置公共读 |

解决方案：

放弃视频功能，专注于文字+图片

在文档中记录探索过程，作为技术储备

经验总结：

多模态 API 的视频功能往往需要单独开通

OSS 文件必须显式设置为 public-read

案例2：参数调节功能的迭代优化
背景：实现“最大生成长度”滑动条，让用户控制回复长度。

版本1：直接截断

实现：直接设置 max_tokens 参数

问题：回复被强行截断，出现不完整句子

示例：设置 50 字，回复在第 50 字处切断

版本2：添加字数提示

实现：在用户消息后自动添加“请用少于X个字回答”

问题：

提示词显示在用户输入框中，影响体验

模型回复中有时会提及“X个字以内”

版本3：隐藏提示词（最终方案）

```python
# 发给模型的消息（带提示词）
model_message = f"{message} 请用少于{max_tokens}个字回答。"
messages.append({"role": "user", "content": model_message})

# 聊天记录显示原始消息
chat_history.append((message, reply))
```

效果：

✅ 用户看不到提示词

✅ 模型按字数限制主动控制回复

✅ 回复自然，无生硬提及

| 版本 | 用户体验 | 回复自然度 | 字数控制 |
|:---|:---|:---|:---|
| 直接截断 | 中 | 低（截断） | 强制 |
| 提示词暴露 | 低 | 中 | 主动 |
| **隐藏提示词** | **高** | **高** | **主动** |

经验总结：

参数调节应该是“主动控制”而非“被动截断”

内部优化不应影响界面展示

模型提示词设计需要平衡效果和自然度

案例3：网络隧道工具探索
背景：需要将本地文件暴露给阿里云 API 下载。

尝试的工具：

| 工具 | 尝试结果 | 问题 |
|:---|:---|:---|
| ngrok | ❌ 失败 | URL 返回 404，转发配置复杂 |
| localtunnel | ⚠️ 部分成功 | 出现密码验证页面，公网访问不稳定 |
最终方案：改用 阿里云 OSS

✅ 文件直接上传到云端

✅ 生成稳定公网 URL

✅ 无需本地服务器

经验总结：

云存储比内网穿透工具更稳定可靠

调试网络问题时要考虑“公网访问链路的完整性”
